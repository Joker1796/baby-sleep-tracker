# Как проверять BabyApp

Приложение — перенос PWA `babySleepTracker` на React Native (Expo SDK 57).
Проверка делится на два уровня: **автоматические проверки** (быстро, без устройства)
и **ручной прогон в приложении** (на симуляторе/телефоне).

---

## 1. Автоматические проверки (без устройства)

Из папки `BabyApp/`:

```bash
# Типы — должно завершиться без ошибок
npx tsc --noEmit

# Юнит-тесты бизнес-логики (60 тестов: age / advisor / sleepAnalyzer / guidance)
npm test

# Сборка бандла — ловит ошибки импортов/резолвинга модулей
npx expo export --platform ios --output-dir /tmp/babyapp-export
```

Ожидаемо: `tsc` молчит, `npm test` → `60 passed`, `expo export` → `Bundled … Exported`.

---

## 2. Запуск приложения

Нужен Node 18+, установленные зависимости (`npm install`).

```bash
npm start          # QR-код: открыть в Expo Go (iOS/Android)
npm run ios        # iOS-симулятор (нужен Xcode)
npm run android    # Android-эмулятор (нужен Android Studio/AVD)
```

- **Проще всего:** установить **Expo Go** на телефон, запустить `npm start`, отсканировать QR.
- Первый запуск дольше — Metro собирает бандл. После — hot-reload.
- Если хочешь переустановить с нуля: `npx expo start -c` (сброс кэша).

При старте видно короткий сплэш-лоадер (инициализация SQLite + AsyncStorage), затем —
онбординг (если профиля ещё нет) или главный экран.

---

## 3. Ручной прогон по сценариям

Пройди по порядку — это покрывает все экраны и ключевую логику.
Иконки времени в форме открывают нативный date/time-picker.

### Онбординг (первый запуск / нет детей)
- [ ] Открывается экран «Режим малыша» с формой.
- [ ] Заполни имя, дату рождения, пол, кормление → «Добавить».
- [ ] После сохранения приложение само переходит на вкладки.
- [ ] **Перезапусти приложение** (закрой полностью и открой) — профиль на месте (значит, SQLite пишет и читает).

### Сегодня (главный экран)
- [ ] Кнопка сна («Уснул»/«Проснулся») создаёт и закрывает интервал; иконка ☀️/🌙 и подпись меняются.
- [ ] Статус-карточка показывает «Спит …» / «Бодрствует …», таймер идёт (обновление раз в 30 сек).
- [ ] Полоса «окна бодрствования» и «время до сна ~…» отображаются, когда малыш бодрствует.
- [ ] Кнопки активностей (Выкладывание, Купание — интервалы; Покакал — точка) логируют события, показывается тост.
- [ ] Переключатель «✨ Авто / 🎛️ Свой» справа в карточке меняет режим (при «Свой» появляется вкладка «Мой режим»).
- [ ] Появляются ситуативные карточки: советник, укладывание (SettlingFlow), поздравление дня и т.п. — по условиям времени/возраста.
- [ ] Крестики на карточках (поздравление, поддержка, подсказки) скрывают их.
- [ ] Блок «Быстрые темы» — тап по тегу разворачивает текст.

> Подсказка для проверки по времени: чтобы увидеть фазы «скоро сон / пора укладывать / ночное
> пробуждение», удобно менять время события в «Истории» или тестировать в разное время суток.

### История
- [ ] Стрелки ‹ › листают дни (вперёд дальше «сегодня» — заблокировано).
- [ ] Сводка дня: всего сна, ночной, дневной + число снов, на животе, покакал.
- [ ] Тап по событию в таймлайне открывает модалку редактирования: меняй время начала/конца, заметку → «Сохранить».
- [ ] «+ Добавить событие» → выбор типа чипами, время, сохранение.
- [ ] Удаление события (в модалке «Удалить» → подтверждение) убирает его.
- [ ] «🗓️ Построить расписание на завтра» переводит на вкладку «Статистика» с раскрытым расписанием.

### Статистика
- [ ] Столбчатый график (react-native-svg): ночь/день по дням, пунктиром — норма.
- [ ] Переключатель «7 дней / 14 дней».
- [ ] Блок «В среднем …» и вердикт по норме (когда есть данные).
- [ ] «Построить расписание»: 24-часовая полоса, поля «Начало/Конец дня» (time-picker), список снов.

### Советы
- [ ] Переключатель «По возрасту / Все возрасты» меняет набор статей.
- [ ] Тап по статье разворачивает текст (абзацы и списки), шеврон меняется вверх/вниз.

### Мой режим (виден при режиме «Свой»)
- [ ] Карточка возрастных норм.
- [ ] Числовые поля (окно бодрствования, число/длительность снов и т.д.) и поля времени сохраняются.
- [ ] Изменения влияют на «Целевые ориентиры» и на расчёты на «Сегодня».

### Настройки
- [ ] Список детей; ✏️ — редактирование, 🗑 — удаление (с подтверждением).
- [ ] «+ Добавить ребёнка» → форма → появляется в переключателе профилей (на «Сегодня» вверху, если детей >1).
- [ ] Тема: «Как в системе / Светлая / Тёмная» — переключается сразу; «Как в системе» реагирует на системную тему устройства.
- [ ] **Экспорт**: открывается системный диалог «Поделиться», сохраняется `.json`.
- [ ] **Импорт**: выбор файла → «Добавить» или «Заменить всё» → данные подтягиваются.

### Мультипрофиль
- [ ] На «Сегодня» переключение чипами между детьми меняет все данные (события, возраст, советы).

### Тёмная тема
- [ ] Переключи на тёмную — цвета фона/карточек/текста/иконок читаемы на всех экранах.

## 4. Что проверять при отладке

- **Данные не сохраняются между запусками** → смотри инициализацию SQLite в `src/db/index.js` и `bootstrap()` в `src/bootstrap.ts`.
- **Экран пустой/белый после сплэша** → ошибка в `bootstrap()` (открой логи Metro в терминале, там будет стек).
- **Красный экран (redbox)** → текст ошибки укажет модуль; чаще всего — рассинхрон версий нативных модулей (лечится `npx expo install --check`).
- **Пикеры даты/времени не открываются** → `@react-native-community/datetimepicker` требует нативной сборки; в Expo Go работает, в кастомном dev-client нужен prebuild.

## 5. Сверка с оригиналом

Эталон поведения — исходный PWA. Запусти его параллельно и сравни:

```bash
cd ../babySleepTracker && npm install && npm run dev
```

Бизнес-логика (`src/logic`, `src/data`) перенесена дословно, поэтому расчёты
(окна сна, советы, статистика, расписание) должны совпадать при одинаковых данных.
