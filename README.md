# Режим малыша — трекер сна ребёнка

Мобильное приложение (iOS/Android) для отслеживания сна и режима младенца с
подсказками на основе возрастных норм: когда укладывать в следующий раз, как
выстроить распорядок, что делать при коротком сне или ночном пробуждении.

Порт PWA (Vue 3) на **React Native / Expo SDK 57**. Вся бизнес-логика и база
знаний перенесены без изменений; UI переписан на нативные компоненты. Данные
хранятся только на устройстве (офлайн), бэкенда нет.

> Подсказки носят информационный характер и не заменяют консультацию педиатра.

## Возможности

- **Сегодня** — трекинг сна в реальном времени: окно бодрствования, прогноз следующего сна, кнопка «Уснул/Проснулся», активности (купание, выкладывание, и т.д.), ситуативные советы, поздравления с «месяцами».
- **История** — события по дням, сводка (ночной/дневной сон, число снов), редактирование и добавление событий.
- **Статистика** — графики сна за 7/14/30 дней с наложением норм, построение расписания на завтра.
- **Советы** — база знаний (~20 статей), фильтрация по возрасту.
- **Мой режим** — настраиваемые параметры сна (окна, число/длительность снов, отбой), переопределяющие авто-расчёт.
- **Настройки** — несколько детей, тема (авто/светлая/тёмная), напоминание «пора укладывать» (локальное уведомление), экспорт/импорт резервной копии (JSON).

## Стек

- React Native `0.86` / React `19` / Expo SDK `57` / TypeScript
- **expo-sqlite** — локальное хранилище (children/events)
- **Zustand** — состояние; **AsyncStorage** — настройки/эфемерные данные
- **React Navigation** (bottom tabs) · **react-native-svg** (графики) · **dayjs** (даты, локаль ru)
- `@expo/vector-icons` (Ionicons), `@react-native-community/datetimepicker`
- **expo-notifications** — локальное напоминание об укладывании; **expo-haptics** — тактильный отклик

## Структура

```
src/
├── data/        база знаний и нормы (чистый JS, портировано из PWA)
├── logic/       движок-советник, расчёты сна (чистый JS + тесты)
├── db/          слой expo-sqlite
├── store/       Zustand-сторы (children, events, settings, settling) + тесты
├── notifications/ локальное напоминание «пора укладывать»
├── time/        реактивные «часы» (тик 30с)
├── utils/       бэкап, хаптика, анимации раскрытия
├── theme/       палитра light/dark, ThemeProvider, общие стили
├── navigation/  bottom-tabs
├── screens/     7 экранов
└── components/  переиспользуемые компоненты
```

## Требования

- Node 18+, npm
- **Android:** Android Studio (Android SDK, platform-tools) — см. [`ANDROID_LOCAL.md`](./ANDROID_LOCAL.md)
- **iOS:** macOS + Xcode + CocoaPods — см. [`IOS_LOCAL.md`](./IOS_LOCAL.md)

> Expo Go **не подходит** — проект на SDK 57 новее публичного Expo Go. Нужен dev-build (`expo run:*`).

## Запуск

```bash
npm install
```

**Android** (устройство по USB с включённой отладкой или эмулятор):
```bash
npx expo run:android
# если телефон переподключали и «Unable to load script»:
adb reverse tcp:8081 tcp:8081
```

**iOS** (симулятор):
```bash
npx expo run:ios
```

Повседневно (приложение уже установлено — нужен только Metro):
```bash
npx expo start --dev-client   # затем открыть приложение на устройстве/в симуляторе
```

## Проверка

```bash
npm run typecheck    # типы (tsc)
npm run lint         # ESLint (eslint-config-expo)
npm test             # юнит-тесты логики и сторов (vitest, src/)
npm run e2e          # e2e-смоук (Maestro, нужно собранное приложение на симуляторе)
```

Те же проверки (кроме e2e) гоняет CI (GitHub Actions) на каждый пуш.
Сценарии ручной проверки — в [`TESTING.md`](./TESTING.md).

## Данные и бэкап

Всё хранится локально в SQLite. В «Настройках» есть экспорт в `.json` (через системный
«Поделиться») и импорт (замена или слияние) — для переноса между устройствами.
